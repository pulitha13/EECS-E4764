Marco Polo

A modern solution to braille tags, Marco ... Polo

Marco Polo is a solution for visually impaired individuals that scans RFID tags to provide voice-based feedback about items. The system stores information on each item's last known location and quantity, offering a more efficient alternative to traditional Braille tags. Using RFID for data storage and audio processing for voice commands, Marco Polo delivers real-time, intelligent responses through a microprocessor and large language model (LLM). This technology enhances item recognition and management, improving the quality of life for visually impaired individuals and offering significant societal and market potential.

Architecture

When the user speaks a command into the mic, such as "What is this?"" or "I want to store/register this XXX", the digital audio is captured by the webapp and sent to the LLM to process based on our customized prompt. It converts the auditory command into a well defined JSON packet and sends it to the ESP, which processes this information and turns on the buzzer to indicate it is ready to R/W onto the RFID tag. Once a tag is within range, it will be read or written to and the ESP sends a completion notification to the LLM which then uses a speaker and replies with a message.

Technical Components

Our physical components are the following:

  1. RFID tags
    RFID tags are passive devices that use radio waves to provide quick, accurate identification and tracking of objects.
  2. Adafruit RFID Breakout Board
    The Adafruit RFID Breakout Board enables easy integration with microcontrollers for reading and writing RFID data, ideal for access control, asset tracking, and identification systems.
  3. Buzzer
    A buzzer emits sound for signaling, providing audio feedback in systems like this where visual cues are impractical.
  4. ESP8266
    The ESP8266 is a low-cost Wi-Fi-enabled microcontroller with built-in TCP/IP stack, ideal for IoT applications.

    5. Our software components are the following:

  5. Python Webapp
    Uses socket programming to interact with on-network ESP, and leverages gradio web interface for audio input and output.
  6. GPT-mini, whisper, and Google TTS
    Technologies used to convert text to and from digital audio as well as interpret this text as a well defined command for the remote device.

For the next prototype for this project we would do a few things different. Firstly we would upgrade the ESP8266 to an ESP32, a more powerful microcontroller for a similar cost. The ESP32 would allow us to expand addtional IO and include a speaker and microphone to allow on device audio capture and response. This digital audio could be streamed to our webapp which will still utilize the same LLM model and TTS to interpret and respond to audio commands. Additionally we could leverage a database that interacts with the webapp to potentially store recent user data in case the user wants to ask things like "where did I put XXX", in which case the web-app could respond with a stored location.
